# graph plugin

High-level graph plugin for Akasha.

## Entry tools

- `graph_plot`
- `graph_stats`
- `plugin.graph`
- `plugin.call graph ...`

## Build `plugin.wasm`

From repository root:

- Windows PowerShell: `./scripts/build_graph_plugin.ps1`
- Bash: `./scripts/build_graph_plugin.sh`

This compiles `plugins/graph/wasm` and copies `plugin.wasm` into `plugins/graph/`.

## Input contract (example)

```json
{
  "tool": "graph_plot",
  "chart": "line",
  "x": ["2026-03-01", "2026-03-02"],
  "series": [
    { "name": "latency", "y": [120, 95] }
  ]
}
```

Daemon tool payload format (auto-generated by Akasha) is also supported:

```json
{
  "tool": "graph_plot",
  "plugin_id": "graph",
  "action": "plot",
  "args": ["line", "120", "95", "110"]
}
```

## Output contract (example)

```json
{
  "ok": true,
  "view": "graph",
  "library": "plotly",
  "figure": {
    "data": [{ "type": "scatter", "mode": "lines", "name": "latency", "x": ["2026-03-01"], "y": [120] }],
    "layout": { "title": "Latency trend" }
  },
  "summary": "Graph generated"
}
```

## Tauri / TUI rendering

- Tauri: native graph panel from `figure` payload.
- TUI: textual stats + mini table fallback.
